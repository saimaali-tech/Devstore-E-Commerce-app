# Screenshots (capture checklist)

Screenshots should be taken **from your machine** after you bring the stack up, so paths and timestamps look authentic on GitHub and LinkedIn.

## What to capture

1. **Home hero** — full-width hero with DevStore branding (`APP_RELEASE=v3` shows the reliability banner).
2. **Shop grid** — product cards with pricing and badges.
3. **Wishlist (V2+)** — wishlist page with at least one saved item.
4. **Order confirmation** — success state with order number; V3 shows the fulfillment timeline panel.

## Suggested filenames

Store under `docs/screenshots/` (gitignored if you prefer large binaries—otherwise commit resized PNGs):

- `01-home.png`
- `02-shop.png`
- `03-wishlist.png`
- `04-order-confirmation.png`

## Tips

- Use a consistent browser width (e.g. 1440×900) for a clean README gallery.
- Redact emails if you use real addresses in forms.
